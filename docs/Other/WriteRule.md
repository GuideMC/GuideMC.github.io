# 本文档编写规范

> 本页面主要讲解如何规范编写本文档。

##  排版

:::tip 排版提示
务必参考该项目：[中文文案排版指北](https://github.com/sparanoid/chinese-copywriting-guidelines/blob/master/README.zh-Hans.md)
:::

## 文本格式

- 中英文、数字与符号：

在中文与英文 / 数字 / 代码块之间使用空格格式化，在段落中间的链接也需要增加空格。

- 规范化专有名词
- 规范使用标题，请按层级使用
- 文章色彩务必中性化规范化，要求简洁明了

## 图片 / 外部资源

- 图片应不带有个人色彩，比如昵称 / 非默认设置等影响初学者的内容
- 除存放在 `docs/assets` 之外，应优先使用其他图床
- 请务必使用 `./` 引用相对路径。使用 `/` 会导致构建时崩溃

## 文档命名

- 务必使用大驼峰命名法

> 大驼峰命名法: 所有单词首字母大写

:::tip 例外
`index.md` 和 `assets` 文件夹
:::

## 如何贡献文档

请先 Fork 本仓库，而后在您的仓库内进行更改，更改完成后 Pull request 至 `main` 分支 (如果修改了构建代码请选择 `dev` 分支)。
